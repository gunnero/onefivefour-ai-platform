# MVP Product Requirements

## Product Summary

OneFiveFour AI Media Platform lets a publishing company manage an AI workforce. The MVP must show the Organization, Departments, AI Employees, Assignments, SOPs, Policies, logs, messages, and performance metrics that make the platform feel like an operating company rather than a prompt runner.

## Primary User

The primary MVP user is a human operator or editor who supervises AI Employees and reviews their work before it reaches a CMS.

## Core Jobs

- Understand the Organization structure.
- See which Departments exist.
- See which AI Employees work in each Department.
- Understand an AI Employee's mission, capabilities, permissions, current Assignments, logs, and performance.
- Create and review Assignments.
- See how SOPs and Company Policies guide work.
- Keep Brain / Model configuration separate from employee identity.

## Functional Requirements

- The system must support at least one Organization.
- The system must support Departments scoped to an Organization.
- The system must support persistent AI Employees scoped to an Organization and Department.
- The system must support Assignments scoped to an Organization, Department, and AI Employee.
- The system must support Assignment Logs.
- The system must support Employee Messages.
- The system must support Employee Performance Metrics.
- The system must support Company Policies.
- The system must support Standard Operating Procedures.
- The system must support mock Brain / Model configuration.
- The system must not call real Brain / Model providers in the first implementation.

## Non-Functional Requirements

- Core records must be auditable.
- Customer data must be organization-scoped.
- Status changes must be explicit.
- Human approval must be required for publishing decisions.
- Terminology must remain consistent across UI, API, database, docs, and specs.

## MVP Success

The MVP succeeds when a user can manage the initial AI company structure and follow an Assignment from creation through logs, messages, review, and completion without relying on real AI provider integration.
